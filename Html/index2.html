CSS en JS en react (Styled Components, Emotion) / Pros y contras frente a CSS tradicional

React

React es una biblioteca de JavaScript de código abierto diseñada para crear interfaces de usuario con el objetivo de facilitar el desarrollo de aplicaciones en una sola página. Es mantenido por Facebook y la comunidad de software libre.

En React se puede utilizar una librería llamada Styled Components, la cual permite escribir estilos CSS directamente en JavaScript.

Styled Components añade una clase única a cada componente para evitar problemas de especificidad. Además, esto asegura que el buscador solo cargue el código necesario para cada página de tu aplicación.

Otra biblioteca muy conocida es Emotion, esta al igual que Styled Components, permite escribir estilos directamente en tus componentes de React, pero destaca por su enfoque en el rendimiento y su capacidad de ser framework agnóstico.

Pasos para el uso de las librerias

1. Para utilizar cualquiera de los dos componentes, primero debes instalar react através del siguiente comando en la terminal, incluyendo el nombre del proyecto que vas a crear:

npx create-react-app


2. Para instalar estas dependencias se debe ejecutar en la terminal de tu proyecto:

Styled components

En este caso se debe ejecutar en la terminal de tu proyecto el comando:

npm install styled-components


Emotion

En este caso se debe ejecutar en la terminal de tu proyecto el comando:

npm install --save @emotion/styled


3. En cada archivo en el que lo quieras utilizar, debes importarlo. En la parte superior del archivo escribe:

Styled components

import styled from "styled-components";

Emotion

import styled from "@emotion/styled";


La forma en la que se agregan los elementos en el proyecto con cualquiera de las 2 librerías tiene bastante similitud a como se agregan al utilizar Angular

Pros y Contras respecto al CSS tradicional

Algunas de las ventajas que tiene usar Styled Components frente al CSS tradicional son:

  • Los nombres de los selectores son los mismos en un ámbito global, lo que permite que el código sea más fácil de leer.
  • Permite utilizar los elementos de CSS directamente dentro del código JavaScript, a su vez esto ayuda a organizar mejor el código al no tener que cambiar de archivo constantemente.
  • Aunque no tiene autocompletado de forma natural, puedes instalar una extesión para que sí lo tengas, al menos para visual studio code.

    • Algunas de las desventajas que tiene usar Styled Components frente al CSS tradicional son:

  • Al utilizar una sintaxis diferente para definir estilo, agrega una complejidad adicional a la base del código.
  • Styled Components a veces pueden ser más lentos que el CSS simple, especialmente en aplicaciones grandes con muchos componentes.
  • Es más difícil ver en las herramientas de desarrollo cuál es cada componente y es más tedioso debuggear(se refiere al proceso de identificar, analizar y corregir errores o "bugs" en un programa de software. Estos errores pueden ser fallos en la lógica del código, problemas de rendimiento, o cualquier comportamiento inesperado que impida que el software funcione correctamente.).
